你会写代码，但第一次写 GPT 时真正卡人的是：每一行之后 shape 变成了啥、mask 加在哪、为什么是 view(B,T,nh,hd) 这个顺序。这页把整条路 从文本到文本 看成一个 tensor 的 shape 变换链："The cat" → tokenize → (B,T) → … → logits → 采样 → "sat"。每行左边是 shape（主角），右边是 mygpt/model.py 里对应的那行代码（锚点）。改顶部超参，所有 shape 实时重算。

00

先钉死这几个维度的含义

整页所有 shape 都由这几个数拼出来。B/T/C 三个最常出现，是流经模型那个 tensor 的轴；另外几个在特定步骤登场。

B
batch_size
一个 batch 几条序列。纯为喂数据并行，模型逻辑跟 B 无关——所有运算在 B 轴上广播。
T
seq_len / block_size
序列里有几个 token。attention 的 T×T 矩阵就来自这根轴，长上下文贵也因为它。
C
n_embd / d_model
每个 token 的向量维度。embedding 后每个 token 是长度 C 的向量；GPT-2 small 是 768。

其余几个在具体步骤出现：nh=n_head、hd=head_dim = C // nh（拆头时）；vocab_size=词表大小（tokenizer 定下来，输出层用）。
注意区分：vocab_size（大写概念，词表维度）和 attention 里的 v（小写，value 向量）是两个完全不同的东西，本页一律把词表维度写成 vocab_size，不用单字母，避免和 v 混。

读 shape 的习惯： 后面每一行你都能在脑子里（或真在代码里）print(x.shape) 对一遍。每行左边那个带颜色的方块，就是把 x.shape 画出来 —— 绿=B 青=T 红=C 紫=vocab_size。

02

放大第 ⑥ 步：attention 里的数字流

管线第 ⑥ 步那行 att = (q @ k.transpose(-2,-1)) * scale 算出的就是 (B,nh,T,T) 矩阵。它到底装了什么数？这里取 1 个 head、T=3、hd=2 的最小例子，把 q@kᵀ → scale → mask → softmax → @v 一步步用真实数字走完（这些数是 JS 现场算的，你可以拿计算器复核）。

三个写代码时最容易错的点： ① masked_fill 在 softmax 之前，填 -inf（softmax 后变 0）；② softmax 的 dim=-1（按行，每行和=1）；③ att @ v 之后 shape 回到 (B,nh,T,hd)，和 q 一样 —— attention 不改 T 和 hd，只重新加权。
03

训练：targets 是 inputs 左移一位

训练时整条 T 并行算，每个位置同时预测它的下一个 token —— T 个监督信号一次拿到。关键就两件事：targets 怎么来、logits 怎么 reshape 去算 loss。

mygpt/model.py · forward(idx, targets)training
logits = self.forward(idx)              # (B, T, vocab_size)
# targets = idx 左移一位：位置 i 的标签 = 位置 i+1 的 token
loss = F.cross_entropy(
    logits.view(-1, logits.size(-1)),  # (B*T, vocab_size) ← 拍平 batch×时间
    targets.view(-1)                  # (B*T,)  ← 对应的标签
)                                       # loss: 标量

为什么能一次算 T 个 loss

因为 causal mask 保证了位置 i 的输出只依赖 ≤i 的输入。所以"位置 i 预测 i+1"这 T 个任务互不干扰，可以并行塞进同一个 forward。下面这张表就是 idx 和 targets 的对应关系：

这就是 causal mask 存在的理由： 训练时 T 个位置同时算，如果位置 i 能看到 i+1，它就直接抄到了标签。第 ② 节那个三角 mask 把未来挡住，才让"并行预测下一个"这件事成立。
04

生成：一次一个 token + KV cache

推理时没有完整序列，自回归地一个个生成。朴素做法每步重算整条；KV cache 把历史的 k,v 存下来，每步新 token 的 q 只有长度 1。点按钮看 T 怎么增长。

朴素 vs KV cache 的 shape 差别inference
# 朴素：每步把已生成的整条都喂进去重算
q,k,v = ...                 # q: (B, nh, T, hd)  —— T 在变长
att   = q @ k.transpose(-2,-1)   # (B, nh, T, T)  全重算，浪费

# KV cache：只算新 token，k,v 从 cache 拼接
q_new = ...                 # q: (B, nh, 1, hd)   ← 只有 1 个新位置
k = torch.cat([k_cache, k_new], dim=2)  # (B, nh, T, hd)
att   = q_new @ k.transpose(-2,-1)    # (B, nh, 1, T)  只算 1 行
